Etapa 1: História da Web

Ao falarmos da história da web é totalmente aceitável que falemos primeiramente da história dos computadores. Afinal, esses são fundamentais para que a web, como conhecemos hoje, fosse possível. Os primeiros computadores, diferentemente da nossa visão atual, eram máquinas gigantescas que tinham como objetivo principal cálculos matemáticos usados para guerras tais como criptografia. Os primeiros computadores que existiram foram o Havard Mark I, Colossus e ENIAC no ano de 1946. A seguir temos três imagens, respectivas, deles.

Assim, essa ficou sendo a primeira geração de computadores. Computadores gigantescos feitos à base de bobinas e valvulas. A segunda geração foi quando trocaram as válvulas por transistores. Isso permitia deixar o computador mais 'portátil'. Para aquela época entenda como portátil conseguir transferir de uma sala para a outra. Como mostra a imagem abaixo

A terceira geração foi permitida mediante a criação dos circuitos integrados que permitiu deixar os computadores ainda menores. A partir dessa tecnologia que foi possível ter os famosos PC's (Personal Computers) ou computadores pessoais.

Enfim, a quarta geração que é a que nós estamos vivendo agora está relacionada à circuitos cada vez menores e mais potêntes.

A comunicação foi melhorando conforme o passar do tempo. Em 1958 por exemplo, foi passado um cabo de um continente para outro possibilitando a comunicação direta entre eles. Obviamente, devido às guerras a forma de comunicação foram ficando cada vez mais eficazes para que se pudesse ter vantagem. Além disso, como formas de comunicação da época temos o telégrafo e o código Morse.

A primeira palavra a trafegar pela internet foi a palavra 'LO'. A comunicação entre duas universidades foi feita e foi enviada a palavra 'LOGIN' de uma para a outra, mas a outra apenas recebeu 'LO'. Ou seja, a primeira conexão à internet e a primeira queda ocorram no mesmo dia. Para aprimorar isso foi criado o TCP/IP deixando as conexões mais estáveis e logo após foi criada a World Wide Web com intuito de não remediar problemas de diversificação demasiada de redes.

  1. Grace Murray Hopper trabalhou na Marinha - Criou COBOL e o termo 'BUG'.
  2. Joseph Carl Robnett Licklider - teorizou a idéia de Cloud.
  3. Robert E. Kahn - criou TCP/IP e primeiro email.
  4. Tim Berners-Lee - considerado o pai da internet, criou o WWW, popularizou HTTP e HTML.
  5. Marc Andreessen - Reinventou os navegadores.
  6. Bill Gates - Fundador da Microsoft, criou o Internet Explorer.
  7. Steve Jobs - Fundador da Apple, popularizou o smartphone.

Etapa 2: O que são Clients

Client é quem consume algo da internet. No momento que estamos utilizando a internet para obter alguma informação somos Clients. Para nos auxiliar nessa missão de obter dados da internet existem os navegadores que servem para interpretar os programas no qual recebemos as informações. Quando solicitamos alguma informação da Web sua resposta vem em HTML, CSS e JavaScript geralemente. O navegador traduz essas informações para que possamos ler o que desejamos.

Os Clients buscam informações de diferentes dispositivos. Quando se busca informação através de um navegaor ou 'Browser' estamos acessando uma Aplicação Web. Ou seja, aplicação web é tudo aquilo que não precisa ser instalado no dispositivo Client. Diferentemente dos aplicativos de celular, por exemplo, que são chamados de dispositivos móveis no qual fazemos a instalação para conseguir consumir o conteúdo desejado.

Etapa 3: O que são Servers?

Um server é tudo que demanda arquivos. É uma máquina que distruibui arquivos Existem vários tipos de servidores tais como de arquivos, serurança, streaming, e-emails e web. Quando temos um conjunto de servidores damos o nome de Data Center. Tudo que envolve computação é feito de duas partes: hardware e Software. O primeiro são as partes física, as peças, e o segundo é a parte lógica, os sistemas de operação e programas.

Quando uma requisição HTTP é requisitada na web, um servidor que responde. Os servidores podem ser tanto local como Web. Um servidor Proxy é um servidor que faz o bloqueio mediante algumas regras. O Firewall faz uma coisa análoga, entretanto fala sobre acessos externos. Quando você tem uma aplicação web que é interna, ou seja, é aplicada apenas para uma rede local, dizemos que a aplicação está na Intranet. Só é acessível para a rede local. DNS - Domain Name Service é um tipo de servidor que é direcional à um IP. É registrado pelo Domínio br. Hospedagem de site é disponibilizar um site para as pessoas acessarem podendo ser local ou não. HTTPS é segura ou seja é criptografada de ponta a ponta. FTP é uma conexão direta com o servidor para você transferir arquivos. File Transfer Protocol.

Etapa 4: Linguagens de Programação

Uma linguagem de progração é um ocnjunto de informaçoes que você insere que servem para comunicação direta com o computador. Atualmente, as linguagens que estão sendo mais usadas no mundo são JavaScript, Python, Java, PHP e C#. As lingugagens rodam em diferentes tipos de lugares. Existem linguagens que rodam do lado do servidor (Server-side) tipo: Python, PHP, C#... Já outras rodam do lado do cliente (Client-side) tais como JavaScript e CSS. Os desenvolvedores que programam linguagens que rodam do lado do servidor são chamados de Back-Ends. Quando a linguagem roda do lado do Cliente chamamos de Front-End.
